React – Componentização Avançada na Prática

Vamos aprender componentização avançada através de um exemplo prático: criaremos uma página de dashboard completa e depois a quebraremos em componentes menores e reutilizáveis. Usaremos Tailwind CSS para estilização e mostraremos como cada pedaço da interface pode se tornar um componente.

  • ✅ Boas práticas de organização de código

1. Página Completa - Dashboard

Vamos começar criando uma página de dashboard completa com tudo em um único componente. Esta é a abordagem "monolítica" que depois refatoraremos.

1.1 Dashboard Monolítico

import React, { useState } from 'react';

function Dashboard() {
  const [user] = useState({

    email: 'joao@email.com',

            </div>

              <img 
                className="w-10 h-10 rounded-full" 
                src={user.avatar} 
                alt={user.name} 
              />

            <h2 className="text-lg font-medium text-gray-900">Pedidos Recentes</h2>

                    Cliente
                  </th>
                  <th className="px-6 py-3 text-left text-xs font-medium text-gray-500 uppercase tracking-wider">
                    Valor
                  </th>
                  <th className="px-6 py-3 text-left text-xs font-medium text-gray-500 uppercase tracking-wider">
                    Status

                </tr>
              </thead>

                  <tr key={order.id}>
                    <td className="px-6 py-4 whitespace-nowrap text-sm text-gray-900">
                      #{order.id}
                    </td>
                    <td className="px-6 py-4 whitespace-nowrap text-sm text-gray-900">

export default Dashboard;
Problema: Este código funciona, mas é difícil de manter, testar e reutilizar. Cada parte da interface está misturada em um único componente.

3. Refatorando em Componentes Menores

3.1 Componentes Básicos

StatusBadge Component
function StatusBadge({ status }) {
  const getStatusStyles = (status) => {
    switch (status) {
      case 'Entregue':
        return 'bg-green-100 text-green-800';
      case 'Em trânsito':
        return 'bg-yellow-100 text-yellow-800';
      case 'Processando':
        return 'bg-blue-100 text-blue-800';
      default:
        return 'bg-gray-100 text-gray-800';
    }
  };

  return (
    <span className={`inline-flex px-2 py-1 text-xs font-semibold rounded-full ${getStatusStyles(status)}`}>
      {status}
    </span>
  );
}

function DataTable({ title, columns, data }) {
  return (
    <div className="bg-white rounded-lg shadow">
      <div className="px-6 py-4 border-b border-gray-200">
        <h2 className="text-lg font-medium text-gray-900">{title}</h2>
      </div>
      <div className="overflow-hidden">
        <table className="min-w-full divide-y divide-gray-200">
          <thead className="bg-gray-50">
            <tr>
              {columns.map((column, index) => (
                <th key={index} className="px-6 py-3 text-left text-xs font-medium text-gray-500 uppercase tracking-wider">
                  {column}
                </th>
              ))}
            </tr>
          </thead>
          <tbody className="bg-white divide-y divide-gray-200">
            {data.map((row, index) => (
              <tr key={index}>
                {Object.values(row).map((cell, cellIndex) => (
                  <td key={cellIndex} className="px-6 py-4 whitespace-nowrap text-sm text-gray-900">
                    {cell}
                  </td>
                ))}
              </tr>
            ))}
          </tbody>
        </table>
      </div>
    </div>
  );
}

4. Dashboard Refatorado

Agora vamos ver como nossa página principal fica muito mais limpa e organizada: 

import React, { useState } from 'react';
import Header from './components/Header';
import UserProfile from './components/UserProfile';
import StatsGrid from './components/StatsGrid';
import DataTable from './components/DataTable';
import StatusBadge from './components/StatusBadge';

function Dashboard() {
  const [user] = useState({
    name: 'João Silva',
    email: 'joao@email.com',
    avatar: 'https://via.placeholder.com/40'
  });

  const [stats] = useState([
    { label: 'Vendas', value: 'R$ 12.500', change: '+15%', color: 'green' },
    { label: 'Usuários', value: '1.234', change: '+8%', color: 'blue' },
    { label: 'Pedidos', value: '89', change: '+23%', color: 'purple' },
    { label: 'Receita', value: 'R$ 45.200', change: '+12%', color: 'orange' }
  ]);

  const [recentOrders] = useState([
    { id: 1, customer: 'Maria Santos', amount: 'R$ 150', status: 'Entregue' },
    { id: 2, customer: 'Pedro Costa', amount: 'R$ 89', status: 'Em trânsito' },
    { id: 3, customer: 'Ana Lima', amount: 'R$ 320', status: 'Processando' }
  ]);

  // Preparar dados para a tabela
  const tableData = recentOrders.map(order => ({
    id: `#${order.id}`,
    customer: order.customer,
    amount: order.amount,
    status: <StatusBadge status={order.status} />
  }));

  return (
    <div className="min-h-screen bg-gray-50">
      <Header title="Dashboard">
        <UserProfile user={user} />
      </Header>

      <main className="max-w-7xl mx-auto px-4 sm:px-6 lg:px-8 py-8">
        <StatsGrid stats={stats} />
        
        <DataTable
          title="Pedidos Recentes"
          columns={['ID', 'Cliente', 'Valor', 'Status']}
          data={tableData}
        />
      </main>
    </div>
  );
}

export default Dashboard;
Resultado: O código agora está muito mais organizado, cada componente tem uma responsabilidade específica e pode ser reutilizado em outras partes da aplicação!

function Card({ header, footer, children, className = '' }) {
  return (
    <div className={`bg-white rounded-lg shadow ${className}`}>
      {header && (
        <div className="px-6 py-4 border-b border-gray-200">
          {header}
        </div>

      <div className="px-6 py-4">
        {children}
      </div>

        <div className="px-6 py-4 border-t border-gray-200 bg-gray-50">

        </div>
      )}
    </div>
  );
}

6.3 Componente com Props Dinâmicas

function Button({ 
  variant = 'primary', 
  size = 'md', 
  children, 
  ...props 
}) {
  const baseClasses = 'font-medium rounded-lg focus:outline-none focus:ring-2 focus:ring-offset-2';
  
  const variants = {
    primary: 'bg-blue-600 text-white hover:bg-blue-700 focus:ring-blue-500',
    secondary: 'bg-gray-600 text-white hover:bg-gray-700 focus:ring-gray-500',
    danger: 'bg-red-600 text-white hover:bg-red-700 focus:ring-red-500'
  };
  
  const sizes = {
    sm: 'px-3 py-1.5 text-sm',
    md: 'px-4 py-2 text-sm',
    lg: 'px-6 py-3 text-base'
  };

  return (
    <button
      className={`${baseClasses} ${variants[variant]} ${sizes[size]}`}
      {...props}
    >
      {children}
    </button>
  );
}

7. Boas Práticas de Componentização

✅ O que Fazer:
  • Componentes pequenos e com responsabilidade única
  • Nomes descritivos para componentes e props
  • Usar TypeScript para tipagem (quando possível)
  • Documentar props complexas com PropTypes
  • Organizar imports de forma lógica
  • Usar composition over inheritance
❌ O que Evitar:
  • Componentes muito grandes (mais de 200 linhas)
  • Props drilling excessivo
  • Lógica de negócio dentro de componentes de UI
  • Nomes genéricos como "Component" ou "Wrapper"
  • Misturar responsabilidades no mesmo componente
  • Hardcoding de dados dentro de componentes

Exercício 2: Lista Reutilizável

Objetivo: Criar um componente de lista que aceite dados dinâmicos e renderização customizada.

  1. Crie um componente List que aceite um array de dados
  2. Permita renderização customizada de cada item
  3. Adicione suporte para lista vazia
Mostrar solução 
function List({ 
  items, 
  renderItem, 
  emptyMessage = "Nenhum item encontrado",
  className = '' 
}) {
  if (items.length === 0) {
    return (
      <div className={`text-center py-8 text-gray-500 ${className}`}>
        {emptyMessage}
      </div>
    );
  }

  return (
    <div className={`space-y-2 ${className}`}>
      {items.map((item, index) => (
        <div key={index}>
          {renderItem(item, index)}
        </div>
      ))}
    </div>
  );
}

// Uso:
const users = [
  { id: 1, name: 'João', email: 'joao@email.com' },
  { id: 2, name: 'Maria', email: 'maria@email.com' }
];

<List
  items={users}
  renderItem={(user) => (
    <div className="p-4 bg-white rounded border">
      <h3>{user.name}</h3>
      <p>{user.email}</p>
    </div>
  )}
/>

Exercício 3: Modal Reutilizável

Objetivo: Criar um componente Modal que use children para conteúdo flexível.

  1. Crie um Modal com backdrop e animação
  2. Use children para o conteúdo
  3. Adicione suporte para fechar com ESC ou clique fora
Mostrar solução 
import { useEffect } from 'react';

function Modal({ isOpen, onClose, children, title }) {
  useEffect(() => {
    const handleEsc = (e) => {
      if (e.key === 'Escape') onClose();
    };
    
    if (isOpen) {
      document.addEventListener('keydown', handleEsc);
      document.body.style.overflow = 'hidden';
    }
    
    return () => {
      document.removeEventListener('keydown', handleEsc);
      document.body.style.overflow = 'unset';
    };
  }, [isOpen, onClose]);

  if (!isOpen) return null;

  return (
    <div className="fixed inset-0 z-50 overflow-y-auto">
      <div className="flex items-center justify-center min-h-screen px-4">
        <div 
          className="fixed inset-0 bg-black bg-opacity-50 transition-opacity"
          onClick={onClose}
        />
        <div className="relative bg-white rounded-lg shadow-xl max-w-md w-full">
          <div className="flex items-center justify-between p-6 border-b">
            <h3 className="text-lg font-medium">{title}</h3>
            <button
              onClick={onClose}
              className="text-gray-400 hover:text-gray-600"
            >
              ✕
            </button>
          </div>
          <div className="p-6">
            {children}
          </div>
        </div>
      </div>
    </div>
  );
}

// Uso:
<Modal isOpen={showModal} onClose={() => setShowModal(false)} title="Confirmação">
