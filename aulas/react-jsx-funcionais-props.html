1. Sintaxe Avançada do JSX

1.1 Regras fundamentais

  • JSX precisa retornar um único elemento pai. Use <> ... </> (Fragment).
  • Expressões JS vão entre chaves { }.
  • Comentários em JSX: {/* ... */}.
  • Atributos usam camelCase: className, onClick, tabIndex.
  • class em HTML vira className em JSX. 
const nome = 'Ana';
function Hello() {
  return (
    <>
      <h1>Olá, {nome}!</h1>
      {/* Renderização condicional com && */}
      {nome === 'Ana' && <p>Que bom ver você aqui!</p>}
    </>
  );
}

1.2 Condicionais: && vs. ternário

Cuidado: valores como 0 são renderizados na tela. Se sua expressão puder virar 0, prefira o ternário. 

const pontos = 0;
<p>
  {/* Isto exibirá 0 na tela */}
  {pontos && 'Você tem pontos'}
</p>

<p>
  {/* Melhor: evita mostrar 0 */}
  {pontos > 0 ? 'Você tem pontos' : null}
</p>

<img src="/logo.png" alt="Logo" />

Segurança: JSX escapa valores por padrão (protege contra XSS). Só use dangerouslySetInnerHTML quando souber exatamente o que está fazendo.

2. Componentes Funcionais com Arrow Functions

Componentes devem ter nome em PascalCase e retornar JSX. 

// Forma concisa (retorno implícito)
const Botao = ({ texto }) => <button>{texto}</button>;

// Com bloco (retorno explícito)
const BotaoGrande = ({ texto }) => {
  return <button className="btn btn-lg">{texto}</button>;
};

export default function App() {
  return (
    <div>
      <Botao texto="Salvar" />
      <Botao texto="Cancelar" />
      <BotaoGrande texto="Continuar" />
    </div>
  );
}

2.1 Desestruturação, valores padrão e rest props 

const Alerta = ({ tipo = 'info', children, ...rest }) => {
  const classe = `alert alert-${tipo}`;
  return <div className={classe} {...rest}>{children}</div>;
};

// Uso:
<Alerta tipo="warning" id="aviso">Cuidado!</Alerta>
<Alerta>Mensagem informativa</Alerta>
Erro comum: não esquecer de return quando usar chaves {'{'}{'}'} no corpo da arrow function.

3. Passando Dados com Props

Props são somente leitura. Você envia dados do componente pai para o filho. 

function Saudacao({ nome = 'Visitante', dia = 'hoje' }) {
  return <h2>Olá, {nome}! Como vai {dia}?</h2>;
}

export default function App() {
  const nomes = ['Maria', 'João', 'Aline'];
  return (
    <>
      <Saudacao nome="Maria" dia="essa manhã" />
      <Saudacao /> {/* Visitante, hoje */}
      {nomes.map(n => <Saudacao key={n} nome={n} dia="agora" />)}
    </>
  );
}

3.1 Passando funções como props

const BotaoAcao = ({ onAcao, children }) => (
  <button onClick={onAcao}>{children}</button>
);

export default function App() {
  const dizerOi = () => alert('Oi!');
  return <BotaoAcao onAcao={dizerOi}>Clique</BotaoAcao>;
}

4. Composição com children

children permite envolver conteúdo e criar layouts reutilizáveis. 

function Card({ children }) {
  return <div className="card">{children}</div>;
}

function CardHeader({ children }) {
  return <div className="card-header">{children}</div>;
}

function CardBody({ children }) {
  return <div className="card-body">{children}</div>;
}

export default function App() {
  return (
    <Card>
      <CardHeader><h2>Título</h2></CardHeader>
      <CardBody><p>Conteúdo enviado como children.</p></CardBody>
    </Card>
  );
}

5. Renderizando Listas (e a importância de key)

Para criar vários elementos a partir de um array, utilize map() e forneça uma key única e estável. 

const itens = ['React', 'Vue', 'Angular'];

export default function App() {
  return (
    <ul>
      {itens.map(framework => (
        <li key={framework}>{framework}</li>
      ))}
    </ul>
  );
}

5.1 Objetos com IDs

const cursos = [
  { id: 101, nome: 'Lógica de Programação' },
  { id: 102, nome: 'Front-end' },
  { id: 103, nome: 'Back-end' },
];

export default function App() {
  return (
    <ol>
      {cursos.map(c => <li key={c.id}>{c.nome}</li>)}
    </ol>
  );
}
Atenção: usar o índice do array como key pode causar bugs em reordenações/remoções. Prefira um identificador estável (ex.: id do item).

6. Hierarquia de Componentes e Boas Práticas 

function Header() {
  return <h1>Meu App</h1>;
}

function Layout({ children }) {
  return <div className="layout">{children}</div>;
}

export default function App() {
  return (
    <Layout>
      <Header />
      <p>Conteúdo principal</p>
    </Layout>
  );
}

Boas práticas:

  • Quebre componentes grandes em menores e reutilizáveis.
  • Evite props drilling (passar props por muitos níveis). Em breve veremos Context API.
  • Nomeie props de eventos com onAlgo e handlers com handleAlgo.
  • Prefira dados imutáveis (não mutar arrays/objetos diretamente).

8. Exercícios Práticos

Exercício 1: Card Componente

Objetivo: criar um componente reutilizável que utilize a prop children.

  1. Implemente o componente Card com uma borda e padding.
  2. Use-o no App para envolver um título e um parágrafo.
  3. Confira o resultado no navegador.
Mostrar solução 
function Card({ children }) {
  const estilo = { border: '1px solid #ddd', padding: '12px', borderRadius: '8px' };
  return <div style={estilo}>{children}</div>;
}

export default function App() {
  return (
    <Card>
      <h3>Título do Card</h3>
      <p>Este é um conteúdo qualquer.</p>
    </Card>
  );
}

Exercício 2: Lista de Itens

Objetivo: renderizar uma lista de elementos a partir de um array.

  1. Crie um array com três ou mais strings.
  2. Use map() para gerar os elementos li.
  3. Defina uma key única para cada item.
Mostrar solução 
const linguagens = ['JavaScript', 'Python', 'Go', 'Rust'];

export default function App() {
  return (
    <ul>
      {linguagens.map(lang => <li key={lang}>{lang}</li>)}
    </ul>

Exercício 3: Saudação com Props

Objetivo: praticar o uso de props criando várias saudações personalizadas.

  1. Reaproveite o componente Saudacao.
  2. Renderize-o enviando nomes diferentes como prop.
  3. Verifique que cada instância exibe a mensagem correta.
Mostrar solução 
function Saudacao({ nome = 'Visitante' }) {
  return <h2>Olá, {nome}!</h2>;
}

export default function App() {
  const nomes = ['Ana', 'Bruno', 'Carla'];

Exercício 4: Botão de Ação

Objetivo: passar uma função como prop e reagir a um clique.

  1. Crie um componente BotaoAcao que recebe onAcao e children.
  2. No App, defina uma função que exibe um alert.
  3. Passe essa função para o botão e teste.
Mostrar solução 
const BotaoAcao = ({ onAcao, children }) => (
  <button onClick={onAcao}>{children}</button>
);

export default function App() {
  const handleClick = () => alert('Ação executada!');
  return <BotaoAcao onAcao={handleClick}>Clique Aqui</BotaoAcao>;
}

Exercício 6: Card com variação de estilo por prop

Objetivo: alterar classe/estilo conforme uma prop.

  1. Crie um componente Badge que recebe tipo (success | warning | info).
  2. Mude a classe do elemento com base em tipo.

9. Referências Rápidas (Resumo)

  • JSX: um pai, expressões em {'{'}{'}'}, className, eventos em camelCase.
  • Props: somente leitura, desestruturação e valores padrão na assinatura.
  • children: composição e layouts reutilizáveis.
  • Listas: use map() e key estável (evite índice).
  • Condicionais: prefira ternário para evitar mostrar 0.
