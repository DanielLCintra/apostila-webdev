React – Eventos, Formulários e Navegação

Nesta aula vamos construir um projeto completo de cadastro de contatos usando Next.js, focando na componentização e navegação entre páginas. Vamos criar uma aplicação que permite adicionar contatos, filtrá-los e navegar para uma página de detalhes passando os dados via URL.

  • ✅ Componentização: Dividir a aplicação em componentes reutilizáveis
  • ✅ Next.js App Router: Navegação entre páginas
  • ✅ Passagem de dados: Enviar dados do contato via URL
  • ✅ Estado em memória: useState para gerenciar contatos
  • ✅ Projeto prático: Aplicação completa de contatos

1. Visão Geral do Projeto

1.1 O que vamos construir

Vamos criar uma aplicação de cadastro de contatos com as seguintes funcionalidades:

  • Página Principal: Formulário para adicionar contatos + lista de contatos + filtro
  • Página de Detalhes: Exibe informações completas de um contato específico
  • Navegação: Clique em um contato na lista → vai para página de detalhes
  • Passagem de Dados: Dados do contato são enviados via URL

1.2 Estrutura de Componentes

A aplicação será dividida em componentes reutilizáveis:

  • ContactForm: Formulário para adicionar novos contatos
  • FilterInput: Campo de filtro para buscar contatos
  • ContactList: Lista que exibe todos os contatos
  • ContactItem: Item individual de cada contato

Arquivo de Estilos Globais (app/globals.css)

O arquivo globals.css deve conter as diretivas do Tailwind CSS:

2.2 Página Principal (app/page.js)

Esta é a página inicial da aplicação. Ela gerencia o estado dos contatos em memória e integra todos os componentes. Quando um contato é clicado na lista, o usuário é direcionado para a página de detalhes. 

"use client";
import { useState } from "react";
import { FilterInput } from "./components/FilterInput";
import { ContactForm } from "./components/ContactForm";
import { ContactList } from "./components/ContactList";

export default function HomePage() {
  const [contacts, setContacts] = useState([]);
  const [filter, setFilter] = useState("");

  function handleSubmit(newContact) {
    setContacts(prev => [...prev, newContact]);
  }

  function handleRemove(id) {
    setContacts(prev => prev.filter(c => c.id !== id));
  }

  // Filtro simples por nome ou email
  const filtered = contacts.filter(c =>
    c.nome.toLowerCase().includes(filter.toLowerCase()) ||
    c.email.toLowerCase().includes(filter.toLowerCase())
  );

  return (
    <div className="min-h-screen bg-gray-200 p-6">
      <div className="max-w-3xl mx-auto space-y-6">
        <header className="flex items-center justify-between">
          <h1 className="text-2xl font-bold text-gray-900">Cadastro de Contatos</h1>
          <FilterInput value={filter} onChange={setFilter} />
        </header>

        <ContactForm onAdd={handleSubmit} />

        <ContactList items={filtered} onRemove={handleRemove} />
      </div>
    </div>
  );
}
Estado em Memória: Os contatos são mantidos apenas em memória usando useState. Quando a página é recarregada, os dados são perdidos, mas isso simplifica o foco na navegação.

ContactForm Component
import { useState } from "react";

export function ContactForm({ onAdd }) {
  const [form, setForm] = useState({ nome: "", email: "", telefone: "" });

  function handleChange(e) {
    const { name, value } = e.target;
    setForm((prev) => ({ ...prev, [name]: value }));
  }

  function handleSubmit(e) {
    e.preventDefault();
    if (!form.nome.trim()) return; // Validação simples
    onAdd({ ...form, id: Date.now() });
    setForm({ nome: "", email: "", telefone: "" });
  }

  return (
    <form onSubmit={handleSubmit} className="bg-white shadow rounded p-4 space-y-4">
      <div>
        <label className="block text-sm font-medium mb-1 text-gray-700">Nome</label>
        <input
          name="nome"
          className="w-full border rounded px-3 py-2 text-gray-900"
          value={form.nome}
          onChange={handleChange}
          required
        />
      </div>

      <div>
        <label className="block text-sm font-medium mb-1 text-gray-700">Email</label>
        <input
          name="email"
          type="email"
          className="w-full border rounded px-3 py-2 text-gray-900"
          value={form.email}
          onChange={handleChange}
        />
      </div>

      <div>
        <label className="block text-sm font-medium mb-1 text-gray-700">Telefone</label>
        <input
          name="telefone"
          className="w-full border rounded px-3 py-2 text-gray-900"
          value={form.telefone}
          onChange={handleChange}
        />
      </div>

      <button type="submit" className="bg-blue-600 hover:bg-blue-700 text-white px-4 py-2 rounded">
        Adicionar Contato
      </button>
    </form>
  );
}

import Link from 'next/link';

export function ContactItem({ contact, onRemove }) {
  return (

          href={`/contact/${contact.id}?nome=${encodeURIComponent(contact.nome)}&email=${encodeURIComponent(contact.email)}&telefone=${encodeURIComponent(contact.telefone)}`} 
          className="font-medium text-gray-900 hover:text-blue-600 transition-colors"
        >
          {contact.nome}
        </Link>
        <p className="text-sm text-gray-600">
          {contact.email} • {contact.telefone}

        onClick={() => onRemove(contact.id)}
        className="text-red-600 hover:text-red-700 px-2 py-1 rounded"
      >
        Excluir
      </button>
    </li>
  );
}

export function ContactList({ items, onRemove }) {
  return (
    <section className="bg-white shadow rounded">
      <div className="px-4 py-3 border-b">
        <h2 className="font-medium text-gray-900">Contatos ({items.length})</h2>
      </div>
      <ul className="divide-y">
        {items.length === 0 ? (
          <li className="p-4 text-gray-500">Nenhum contato encontrado</li>
        ) : (
          items.map((c) => (
            <ContactItem key={c.id} contact={c} onRemove={onRemove} />
          ))
        )}
      </ul>
    </section>
  );
}

3. Next.js App Router - Navegação entre Páginas

3.1 Estrutura de Pastas do App Router

O Next.js App Router usa uma estrutura baseada em pastas. Cada pasta representa uma rota: 

app/
├── page.js              # Página inicial (/)
├── contact/
│   └── [id]/
│       └── page.js      # Página de detalhes (/contact/[id])
└── layout.js            # Layout compartilhado
Nota: O [id] na pasta /contact/[id] é um parâmetro dinâmico que será capturado pelo componente da página.

3.2 Componente Link do Next.js

O componente Link do Next.js permite navegação sem recarregar a página, mantendo a experiência SPA (Single Page Application). 

import Link from 'next/link';

// Exemplo de uso no ContactItem
<Link 
  href={`/contact/${contact.id}`} 
  className="font-medium hover:text-blue-600 transition-colors"
>
  {contact.nome}
</Link>
Importante: Sempre use Link do Next.js em vez de <a> para navegação interna. O <a> recarrega a página, perdendo o estado da aplicação.

3.3 Página de Detalhes do Contato (app/contact/[id]/page.js)

Esta página exibe os detalhes de um contato específico. Como não temos persistência, vamos passar os dados do contato via URL usando query parameters. 

// app/contact/[id]/page.js
"use client";
import { useParams, useRouter, useSearchParams } from 'next/navigation';

export default function ContactDetailPage() {
  const params = useParams();
  const router = useRouter();
  const searchParams = useSearchParams();

  // Recebe os dados do contato via query parameters
  const contact = {
    id: parseInt(params.id),
    nome: searchParams.get('nome'),
    email: searchParams.get('email'),
    telefone: searchParams.get('telefone')
  };

  return (
    <div className="min-h-screen bg-gray-50 p-6">
      <div className="max-w-2xl mx-auto">
        <div className="bg-white shadow rounded-lg p-6">
          <div className="flex items-center justify-between mb-6">
            <h1 className="text-2xl font-bold text-gray-900">Detalhes do Contato</h1>
            <button
              onClick={() => router.back()}
              className="bg-gray-100 hover:bg-gray-200 text-gray-900 px-4 py-2 rounded"
            >
              ← Voltar
            </button>
          </div>

          <div className="space-y-4">
            <div>
              <label className="block text-sm font-medium text-gray-700 mb-1">Nome</label>
              <p className="text-gray-900">{contact.nome}</p>
            </div>

            <div>
              <label className="block text-sm font-medium text-gray-700 mb-1">Email</label>
              <p className="text-gray-900">{contact.email}</p>
            </div>

            <div>
              <label className="block text-sm font-medium text-gray-700 mb-1">Telefone</label>
              <p className="text-gray-900">{contact.telefone}</p>
            </div>
          </div>
        </div>
      </div>
    </div>
  );
}
Fluxo de Passagem de Dados:
  1. Usuário adiciona contatos na página principal (estado em memória)
  2. Clica em um contato → dados são passados via URL (query parameters)
  3. Página de detalhes recebe os dados via useSearchParams
  4. Exibe as informações completas do contato
  5. Botão "Voltar" retorna à lista

3.4 Passagem de Parâmetros via URL

Existem várias formas de passar dados entre páginas via URL:

Parâmetros de Rota ([id])
// Pasta: /contact/[id]
// URL: /contact/123
// Captura: const params = useParams(); // params.id = "123"

<Link href="/contact/123">Ver Contato</Link>
Query Parameters (?search=termo)
// URL: /contacts?search=joao&filter=active
import { useSearchParams } from 'next/navigation';

function ContactList() {
  const searchParams = useSearchParams();
  const search = searchParams.get('search'); // "joao"
  const filter = searchParams.get('filter'); // "active"

  function updateSearch(newSearch) {
    const params = new URLSearchParams(searchParams);
    params.set('search', newSearch);
    router.push(`/contacts?${params.toString()}`);
  }
}
Navegação Programática
import { useRouter } from 'next/navigation';

function LoginForm() {

5. Exercícios

Exercício 1: Formulário de Login com Navegação

Objetivo: Criar um formulário de login que navega para uma página de dashboard após sucesso.

  1. Crie campos de email e senha controlados
  2. Valide email (regex) e senha (mín. 6 chars)
  3. Mostre mensagens de erro condicionalmente
  4. Após login bem-sucedido, navegue para /dashboard
Mostrar solução 
"use client";
import { useState } from "react";
import { useRouter } from 'next/navigation';

export default function LoginForm() {
  const [form, setForm] = useState({ email: "", senha: "" });
  const [errors, setErrors] = useState({});
  const router = useRouter();

  function validate(v) {
    const e = {};
    if (!/^[^@\s]+@[^@\s]+\.[^@\s]+$/.test(v.email)) e.email = "Email inválido";
    if ((v.senha || '').length < 6) e.senha = "Mínimo 6 caracteres";
    return e;
  }

  function handleSubmit(ev) {
    ev.preventDefault();
    const e = validate(form);
    setErrors(e);
    if (Object.keys(e).length) return;
    
    // Simula login bem-sucedido
    router.replace('/dashboard');
  }

  return (
    <form onSubmit={handleSubmit} className="space-y-3">
      <div>
        <label className="block text-sm">Email</label>
        <input
          className="border rounded px-3 py-2 w-full"
          value={form.email}
          onChange={(e) => setForm((p) => ({ ...p, email: e.target.value }))}
        />
        {errors.email && <p className="text-sm text-red-600">{errors.email}</p>}
      </div>
      <div>
        <label className="block text-sm">Senha</label>
        <input
          type="password"
          className="border rounded px-3 py-2 w-full"
          value={form.senha}
          onChange={(e) => setForm((p) => ({ ...p, senha: e.target.value }))}
        />
        {errors.senha && <p className="text-sm text-red-600">{errors.senha}</p>}
      </div>
      <button className="bg-blue-600 hover:bg-blue-700 text-white px-4 py-2 rounded">
        Entrar
      </button>
    </form>
  );
}
