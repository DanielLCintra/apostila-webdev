Operadores

Nesta aula, exploramos os diferentes tipos de operadores em JavaScript:

  • Operadores Aritméticos: Para cálculos matemáticos (+, -, *, /, %, **)
  • Operadores de Incremento/Decremento: Para aumentar/diminuir valores (++, --)
  • Operadores de Atribuição: Para atribuir valores (=, +=, -=, *=, /=, %=, **=)
  • Operadores Relacionais: Para comparações (==, ===, !=, !==, >, <, >=, <=)
  • Operadores Lógicos: Para condições lógicas (&&, ||, !)
  • Operador Ternário: Para expressões condicionais concisas (? :)
  • Operadores Modernos: Coalescência nula (??), Encadeamento opcional (?.)
  • Operador Spread: Para expandir iteráveis (...)
  • Operadores Bit a Bit: Para manipulação binária (&, |, ^, ~, <<, >>)


Os operadores são elementos fundamentais na programação JavaScript que permitem realizar operações com valores e variáveis. Nesta aula, vamos explorar os diversos tipos de operadores disponíveis em JavaScript e como utilizá-los no desenvolvimento do projeto TaskMaster.

Operadores Aritméticos

Usados para realizar operações matemáticas básicas entre valores numéricos.


let a = 10;
let b = 3;

// Operações básicas
let soma = a + b;         // 13
let subtracao = a - b;    // 7
let multiplicacao = a * b; // 30
let divisao = a / b;      // 3.3333...
let modulo = a % b;       // 1 (resto da divisão)
let exponenciacao = a ** b; // 1000 (10³)

- Subtração

Operadores de Incremento e Decremento

Aumentam ou diminuem um valor em uma unidade.


let contador = 0;

// Pré-incremento: incrementa e depois retorna
let preIncremento = ++contador; // contador = 1, preIncremento = 1

// Pós-incremento: retorna e depois incrementa
contador = 0; // Resetando contador
let posIncremento = contador++; // posIncremento = 0, contador = 1

// Pré-decremento: decrementa e depois retorna
contador = 2; // Definindo contador
let preDecremento = --contador; // contador = 1, preDecremento = 1

// Pós-decremento: retorna e depois decrementa
contador = 2; // Resetando contador
let posDecremento = contador--; // posDecremento = 2, contador = 1
++x Pré-incremento
x++ Pós-incremento
--x Pré-decremento
x-- Pós-decremento

Operadores de Atribuição

Atribuem valores a variáveis, combinando a atribuição com operações.


let x = 10;

x += 5;  // x = x + 5 -> 15
x -= 3;  // x = x - 3 -> 12
x *= 2;  // x = x * 2 -> 24
x /= 4;  // x = x / 4 -> 6
x %= 4;  // x = x % 4 -> 2
x **= 3; // x = x ** 3 -> 8
= Atribuição básica

-= Atribuição de subtração
*= Atribuição de multiplicação
/= Atribuição de divisão
%= Atribuição de módulo

Operadores Relacionais (Comparação)

Comparam valores e retornam um valor booleano (true ou false).


let a = 10;
let b = 5;
let c = "10";

// Operadores de comparação tradicionais
console.log(a > b);   // true
console.log(a < b);   // false
console.log(a >= b);  // true
console.log(a <= b);  // false

// Operadores de igualdade
console.log(a == b);   // false (valores diferentes)
console.log(a == c);   // true (valores iguais, tipos diferentes)
console.log(a === c);  // false (valores iguais, mas tipos diferentes)
console.log(a != b);   // true (valores diferentes)
console.log(a !== c);  // true (mesmo valor, mas tipos diferentes)
Operador Descrição Exemplo Resultado
== Igual a (compara valor) 10 == "10" true
=== Estritamente igual (compara valor e tipo) 10 === "10" false
!= Diferente de (compara valor) 10 != "10" false
!== Estritamente diferente (compara valor e tipo) 10 !== "10" true
> Maior que 10 > 5 true
< Menor que 10 < 5 false
>= Maior ou igual a 10 >= 10 true
<= Menor ou igual a 10 <= 5 false

Operadores Lógicos

Combinam expressões booleanas e retornam um valor booleano.


let verdadeiro = true;
let falso = false;

// AND lógico (&&): retorna true se ambos os operandos forem true
console.log(verdadeiro && verdadeiro); // true
console.log(verdadeiro && falso);      // false
console.log(falso && falso);           // false

// OR lógico (||): retorna true se pelo menos um operando for true
console.log(verdadeiro || verdadeiro); // true
console.log(verdadeiro || falso);      // true
console.log(falso || falso);           // false

// NOT lógico (!): inverte o valor de verdade
console.log(!verdadeiro); // false
console.log(!falso);      // true

Operador Ternário

Oferece uma forma concisa de escrever condicionais simples.


let idade = 18;
let status = idade >= 18 ? "Adulto" : "Menor de idade";
console.log(status); // "Adulto"

// Exemplo aninhado (não recomendado - difícil leitura)
let faixaEtaria = idade < 12 ? "Criança" 
                : idade < 18 ? "Adolescente" 
                : idade < 60 ? "Adulto" 
                : "Idoso";
console.log(faixaEtaria); // "Adulto"
condição ? expressão se verdadeiro : expressão se falso

O operador ternário é muito útil para atribuições condicionais simples, mas deve ser evitado em casos complexos para manter a legibilidade do código.

Operador de Coalescência Nula (??)

Retorna o operando do lado direito quando o operando do lado esquerdo é null ou undefined.


let valor1 = null;
let valor2 = undefined;
let valor3 = 0;         // 0 é um valor válido, não null nem undefined
let valor4 = "";        // String vazia é um valor válido
let valor5 = false;     // false é um valor válido

let resultado1 = valor1 ?? "Valor padrão"; // "Valor padrão"
let resultado2 = valor2 ?? "Valor padrão"; // "Valor padrão"
let resultado3 = valor3 ?? "Valor padrão"; // 0
let resultado4 = valor4 ?? "Valor padrão"; // ""
let resultado5 = valor5 ?? "Valor padrão"; // false

Diferença entre ?? e ||

Enquanto o operador || considera valores "falsy" como false, 0, "", NaN e null/undefined como merecedores de fallback, o operador ?? só aciona o fallback para null e undefined.

Expressão Resultado com ?? Resultado com ||
null ?? "fallback" "fallback" "fallback"
undefined ?? "fallback" "fallback" "fallback"
0 ?? "fallback" 0 "fallback"
"" ?? "fallback" "" "fallback"
false ?? "fallback" false "fallback"

Operador de Encadeamento Opcional (?.)

Permite ler propriedades aninhadas de objetos sem gerar erro se algumas forem undefined ou null.


let usuario = {
    nome: "João",
    endereco: {
        rua: "Av. Principal",
        numero: 123
    }
};

let usuarioSemEndereco = {
    nome: "Maria"
};

// Com encadeamento opcional
let ruaJoao = usuario?.endereco?.rua; // "Av. Principal"
let ruaMaria = usuarioSemEndereco?.endereco?.rua; // undefined (sem erro)

// Sem encadeamento opcional causaria erro:
// let ruaMariaErro = usuarioSemEndereco.endereco.rua; // TypeError

O operador ?. verifica se cada nível existe antes de prosseguir, prevenindo erros de execução.

Operador Spread (...)

Permite expandir um iterável (array, objeto, string) em locais onde múltiplos elementos são esperados.


// Em arrays
let numeros1 = [1, 2, 3];
let numeros2 = [4, 5, 6];
let todoNumeros = [...numeros1, ...numeros2]; // [1, 2, 3, 4, 5, 6]

// Cópia de arrays (cópia superficial)
let original = [1, 2, 3];
let copia = [...original];
copia.push(4); // Não afeta o array original

// Em objetos
let pessoa = {
    nome: "Ana",
    idade: 30
};

let pessoaComEndereco = {
    ...pessoa, // Copia todas as propriedades de pessoa
    endereco: "Rua das Flores, 123"
};

// Substituição de propriedades
let configuracaoPadrao = {
    tema: "claro",
    fonte: "Arial",
    tamanho: 12
};

let preferenciasUsuario = {
    ...configuracaoPadrao,
    tema: "escuro" // Substitui o tema
};

Casos de uso comuns do operador spread:

  • Concatenação de arrays: Unir múltiplos arrays de forma simples
  • Clonagem superficial: Criar cópias de arrays ou objetos (apenas primeiro nível)
  • Passar parâmetros: Expandir um array para passar elementos individuais como argumentos
  • Mesclagem de objetos: Combinar propriedades de múltiplos objetos
  • Conversão: Transformar strings e outros iteráveis em arrays

Operadores Bit a Bit

Realizam operações em representações binárias dos valores.


let num1 = 5;  // 101 em binário
let num2 = 3;  // 011 em binário

console.log(num1 & num2);   // 1 (001 em binário) - AND
console.log(num1 | num2);   // 7 (111 em binário) - OR
console.log(num1 ^ num2);   // 6 (110 em binário) - XOR
console.log(~num1);         // -6 (inverte todos os bits e soma 1) - NOT
console.log(num1 << 1);     // 10 (1010 em binário) - Deslocamento à esquerda
console.log(num1 >> 1);     // 2 (010 em binário) - Deslocamento à direita

AND bit a bit (&)

5 1 0 1
3 0 1 1
resultado 0 0 1 = 1

OR bit a bit (|)

5 1 0 1
3 0 1 1
resultado 1 1 1 = 7

        return passaConcluida && passaPrioridade;
    });
};

Desafios para Praticar

Coloque em prática o que você aprendeu resolvendo os seguintes desafios.

Desafio 1: Cálculo de Preço

Crie uma função para calcular o preço total de uma tarefa considerando o preço base e o número de horas necessárias para completá-la. Cada hora adicional custa 10% do preço base.

Ver solução 

function calcularPrecoTotal(precoBase, horas) {
    // Calcula o preço da hora adicional (10% do preço base)
    let precoHoraAdicional = precoBase * 0.1;
    
    // Calcula o preço total: preço base + (horas * preço hora adicional)
    let precoTotal = precoBase + (horas * precoHoraAdicional);
    
    return precoTotal;
}

// Exemplos de uso:
let preco1 = calcularPrecoTotal(100, 5); // 100 + (5 * 10) = 150
console.log("Preço total para 5 horas:", preco1);

let preco2 = calcularPrecoTotal(200, 3); // 200 + (3 * 20) = 260
console.log("Preço total para 3 horas:", preco2);

Desafio 2: Verificação de Atraso

Crie uma função que verifique se uma tarefa está atrasada, comparando a data de vencimento com a data atual. Utilize operadores de comparação.

Ver solução 

function verificarAtraso(dataVencimento) {
    // Cria um objeto Date para a data atual
    let hoje = new Date();
    
    // Converte a string de data em um objeto Date
    let dataLimite = new Date(dataVencimento);
    
    // Compara as datas - true se estiver atrasada, false caso contrário
    return hoje > dataLimite;
}

// Exemplos de uso:
let atrasada = verificarAtraso("2023-01-01");
console.log("A tarefa está atrasada?", atrasada); // Provavelmente true

let emDia = verificarAtraso("2026-01-01");
console.log("A tarefa está atrasada?", emDia); // Provavelmente false

Desafio 3: Incremento de Prioridade

Crie uma função para incrementar a prioridade de uma tarefa até um valor máximo de 5.

Ver solução 

function aumentarPrioridade(prioridadeAtual) {
    // Usando operador de pré-incremento
    let novaPrioridade = ++prioridadeAtual;
    
    // Usando operador ternário para limitar o valor máximo
    novaPrioridade = novaPrioridade > 5 ? 5 : novaPrioridade;
    
    return novaPrioridade;
}

// Exemplos de uso:
let prioridade1 = 3;
console.log("Prioridade original:", prioridade1);
console.log("Nova prioridade:", aumentarPrioridade(prioridade1));

let prioridade2 = 5;
console.log("Prioridade original:", prioridade2);
console.log("Nova prioridade:", aumentarPrioridade(prioridade2)); // Deve permanecer 5

Desafio 4: Status da Tarefa

Crie uma função que retorne o status de uma tarefa baseado em sua conclusão, utilizando o operador ternário.

Ver solução 

function statusTarefa(concluida) {
    // Usando operador ternário para determinar o status
    return concluida ? "Concluída" : "Pendente";
}

// Exemplos de uso:
console.log("Status da tarefa 1:", statusTarefa(true));  // Concluída
console.log("Status da tarefa 2:", statusTarefa(false)); // Pendente

Desafio 5: Tarefa Importante

Crie uma função que retorne se uma tarefa é importante com base em sua prioridade e prazo. Uma tarefa é importante se tiver prioridade maior que 3 OU estiver a menos de 2 dias do prazo.

Ver solução 

function tarefaImportante(prioridade, diasParaPrazo) {
    // Usa operador lógico OR (||) para verificar as condições
    return prioridade > 3 || diasParaPrazo < 2;
}

// Exemplos de uso:
console.log("Tarefa com prioridade 4 e 5 dias para o prazo é importante?", 
            tarefaImportante(4, 5)); // true, pela prioridade

console.log("Tarefa com prioridade 2 e 1 dia para o prazo é importante?", 
            tarefaImportante(2, 1)); // true, pelo prazo

console.log("Tarefa com prioridade 2 e 5 dias para o prazo é importante?", 
            tarefaImportante(2, 5)); // false, nem prioridade nem prazo
